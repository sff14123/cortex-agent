# Indexing Policy

절대 일상적인 파일 동기화나 유령 노드 삭제를 위해 `pc_reindex{force: true}`를 호출하지 마십시오. 
증분 인덱싱(`force: false`)이 추가, 변경, 삭제(유령 노드)를 완벽하게 감지하고 1초 만에 최적화하므로 일상적 워크플로우에 완벽히 부합합니다.
`force: true`는 전체 DB를 강제 포맷 및 해시 무시 후 초기화하므로 수 분이 소요되는 파괴적 작업입니다.

[단, force: true가 불가피한 3가지 예외 상황]
1. **파서/스키마 로직 업데이트**: 에이전트가 새로운 언어 파서를 추가하거나 그래프 DB 스키마를 수정했을 때. (유저 파일 내용(Hash)은 그대로여서 증분 인덱싱이 무시해 버리지만, 데이터를 새로 추출하기 위해 강제로 덮어씌워야 함)
2. **임베딩 모델 교체**: 벡터 엔진 모델(Qwen 0.6B -> 최신 모델)이 변경되어 기존 768차원 벡터들이 새 모델 치수와 호환되지 않아 모두 재계산해야 할 때.
3. **재앙 복구**: SQLite / Faiss 인덱스가 심각하게 오염되어 꼬였을 때.
