# 프로젝트 절대 규칙 (Control Tower)

에이전트의 모든 동작은 이 파일의 'MCP-First' 대전제와 세부 규칙을 따릅니다.

## 1. [MCP-First] 모든 동작의 기본
- 모든 작업의 시작은 `cortex-mcp`를 통한 지식 탐색(`pc_memory_search_knowledge`)이다.
- 분석 및 컨텍스트 파악 시 기본 도구보다 MCP 도구(`pc_run_pipeline`, `pc_capsule` 등)를 최우선으로 사용한다.
- **인프라 우선 수복(Maintenance-First)**: MCP 도구 실행 중 에러가 발생하면 기본 도구로 우회하는 것을 금지한다. 에러의 원인이 스크립트 결함이나 환경 문제라면, 해당 인프라를 먼저 수복한 뒤 기존 작업을 재개해야 한다.
- 작업 기록 및 핵심 지식(커밋/PR/프로토콜 등)은 반드시 `memories` DB와 연동하여 관리하며 실시간으로 동기화한다.

## 1-1. [Autonomous Search & Solve] 자율 진화 및 탐색
에이전트는 사용자의 별도 명령 없이 다음을 자율적으로 수행한다:
- **지식 자산화(SCL)**: 해결이 어려웠던 버그나, 타 모델이 빠졌던 함정을 돌파한 경우 그 방법(Actionable Protocol)을 `learning.md` 규칙에 따라 `success_pattern` 카테고리를 DB에 즉시 저장한다.
- **선행 지식 검색**: 작업 초기 또는 원인 불명 에러 발생 시, `pc_memory_search_knowledge`를 사용하여 이전에 도출된 통찰이나 SCL을 스스로 조회하여 문제에 접근한다.
- 분석 캡슐(`pc_capsule`, `pc_run_pipeline`)의 지속적 활용.
## 2. [Safety & Quality] 안전 및 품질
- 사용자의 명시적 승인 없는 파일 삭제 및 대규모 로직 변경을 엄격히 금지한다.
- 대량 파일 처리(>3개) 또는 반복 작업 시 반드시 스크립트(`.agents/scripts/`)를 생성하여 위임한다.
- 오류 발생 시 로직 수정 전, 의존성 및 실행 환경 설정부터 우선 점검한다.

## 3. [Layered Diagnostics] 계층형 디버깅 원칙
- 코드 수정 후에도 동일한 오류 메시지가 반복되는 것은 '변경 사항이 런타임에 반영되지 않음'을 의미한다.
- 이 경우 코드 로직 수정보다 **프로젝트 언어 및 프레임워크별 환경 레이어 및 캐시 초기화**를 최우선으로 수행한다.
- **점검 대상**: 언어별 바이트코드 캐시, 빌드 파이프라인 아티팩트, 패키지 의존성 무결성, 그리고 런타임 프로세스 메모리 등.
- 세부적인 계층별 진단 절차 및 환경별 조치 명령은 `.agents/rules/diagnostics.md`의 프로토콜을 따른다. (Trigger: `model_decision`)
- 2회 이상 동일 증상 반복 시, 반드시 실행 환경과 의존성 무결성을 먼저 점검한 뒤 로직 수정을 재개한다.

## 4. [Common Policy] 공통 정책
- **언어 정책**: 모든 아티팩트 및 보고서, 승인 허가 문구 등은는 반드시 **한국어**로 작성하며, 이모지 사용을 금지한다.
- **실행 보고**: 모든 `run_command` 실행 직전에 해당 명령이 수행할 작업과 목적을 사용자에게 명확히 보고한다.
- **보고 원칙**: 모든 응답의 마지막에는 사용한 **Skill**과 **MCP 도구** 명칭을 짧게 표기한다. 
  - **단, 반드시 응답 전체의 가장 마지막에 1회만 기재한다 (Single Output 원칙).**
  - **Skill**: `cortex-mcp` DB에 인덱싱된 실제 스킬 목록 중 **현재 턴(Turn)**에서 직접 활용된 것만 표기.
  - **MCP**: **현재 턴(Turn)**에서 도구가 성공적으로 호출되어 사용된 MCP 서버만 표기.
  - 해당 사항이 없으면 `none`으로만 표기한다.

## 5. [Visibility & Terminal Control] 가시성 및 실행 제어
- **백그라운드 은폐 금지**: 10초 이상 소요되거나 대량의 로그(진행률, 타임아웃 등)가 발생하는 모든 명령어(`Running command` 포함)를 **에이전트가 백그라운드로 무단 실행하여 사용자에게 숨기는 것을 엄격히 금지한다.**
- **실행 과정 투명성 보장**: 모든 실행 과정은 사용자 시스템의 `stdout`을 통해 실시간으로 관측 가능해야 하며, 에이전트는 이를 무단으로 비동기 은폐하지 않는다.
- **사용자 통제권 보장**: 에이전트는 어떠한 경우에도 `run_command`를 통해 사용자의 요청이나 작업을 자동 수행하지 않는다. 모든 명령어는 사용자가 직접 검토하고 로컬 터미널에서 수동으로 실행할 수 있도록 **채팅창에 완성된 코드블록 형태(Copy/Paste 전용)로만 제공**해야 한다.
- **직접 실행 금지**: 예외 상황 없이 모든 실행 명령은 사용자에게 텍스트로 전달하며, 에이전트가 임의로 터미널을 호출하여 결과를 도출하는 행위를 엄격히 금지한다.
