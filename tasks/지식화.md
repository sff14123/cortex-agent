// turbo-all

1. 현재 세션에서 수행한 작업을 다음 5개 항목으로 구조화한다:
   - **Recent requests**: 사용자의 최근 핵심 요청 (최대 3건)
   - **Pending work**: 미완성이거나 "다음에" 언급된 작업
   - **Key files**: 작업 중 수정/참조한 파일 경로
   - **Breakthrough**: 핵심 돌파 로직 (있는 경우에만)
   - **Current state**: 마지막 작업 상태
2. 위 항목 중 `learning.md`의 기록 판단 기준(환경 레이어 오류, 모델 함정 돌파 등)에 해당하는 내용이 있으면 Actionable 템플릿 4항목으로 정제한다.
   - 해당하는 내용이 없으면 "기록 대상 없음"으로 보고하고 워크플로를 종료한다.
3. 기존 DB에 동일하거나 유사한 키가 있는지 `pc_memory_search_knowledge`로 확인한다.
   - 유사 키가 존재하면 `pc_memory_consolidate`로 기존 키와 병합하여 파편화를 방지한다.
   - 유사 키가 없으면 `pc_memory_write`로 `category: success_pattern` (또는 insight, architecture)으로 신규 생성한다.
4. **[중요: 물리 백업]** DB 저장과 동시에 `.agents/docs/[category]/[key].md` 경로에 해당 내용을 파일로 저장한다. (재인덱싱 시 유실 방지용)
5. 저장 완료 후 "지식 ID: [key]로 영속화 및 파일 백업 완료" 형식으로 보고한다.
